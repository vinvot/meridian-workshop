Known Gaps & Observations
Tasks API not implemented. The frontend calls getTasks(), createTask(), deleteTask(), and toggleTask(), but the backend has no corresponding endpoints — these return 404 at runtime. Tasks currently fall back to mock data from useAuth.
No persistent database. All data is served from static JSON files loaded in memory at startup. purchase_orders.json is the only writable store; it is not persisted across server restarts. A production deployment would require a relational database (PostgreSQL recommended).
No authentication layer. The useAuth composable provides mock user data. There is no JWT, session, or OAuth flow — all API routes are open. A real deployment must add an auth middleware to the FastAPI backend.
Spending endpoints have no filter support. /api/spending/* endpoints do not accept warehouse or category parameters, unlike the other API groups. Filtering on the Finance view is therefore only cosmetic.
